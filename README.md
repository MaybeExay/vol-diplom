# VolunteerHub — Платформа для волонтёрских задач

Веб-приложение для оптимизации работы волонтёрских задач на стеке React + Node.js + SQLite.

## Возможности

### Роли пользователей:
- **Гость** — просмотр главной страницы, событий, участников (ограниченный доступ)
- **Участник** — участие в событиях, редактирование своего профиля
- **Куратор** — создание событий и постов, просмотр участников событий
- **Администратор** — полный доступ: редактирование/удаление любых профилей и постов, управление событиями

### Функционал:
- Регистрация и авторизация (JWT)
- Лента новостей с постами (с изображениями до 5МБ, до 3 фото в посте)
- Каталог событий с фильтрацией и поиском
- Подробная страница события с возможностью участия
- Профили пользователей с редактированием
- Админ-панель для управления контентом

## Быстрый старт

### 1. Установка зависимостей
```bash
npm install
```

### 2. Запуск приложения

**Вариант 1: Одновременный запуск сервера и клиента (рекомендуется)**
```bash
npm run dev
```

**Вариант 2: Раздельный запуск**
```bash
# Терминал 1 — сервер (порт 5000)
npm run start:server

# Терминал 2 — клиент (порт 3000)
npm start
```

## API Endpoints

### Авторизация
- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход
- `GET /api/auth/me` — проверка токена

### Пользователи
- `GET /api/users` — список всех пользователей
- `GET /api/users/:id` — профиль пользователя
- `PUT /api/users/me` — обновить свой профиль
- `PUT /api/users/:id` — обновить профиль (админ)
- `DELETE /api/users/:id` — удалить пользователя (админ)

### События
- `GET /api/events` — список событий (фильтры: status, search)
- `GET /api/events/:id` — подробная информация о событии
- `POST /api/events` — создать событие (куратор, админ)
- `PUT /api/events/:id` — обновить событие (куратор, админ)
- `DELETE /api/events/:id` — удалить событие (админ)
- `POST /api/events/:id/participate` — записаться на событие
- `DELETE /api/events/:id/participate` — отменить участие
- `GET /api/events/:id/participants` — участники события (куратор, админ)

### Посты
- `GET /api/posts` — список постов
- `POST /api/posts` — создать пост (куратор, админ)
- `DELETE /api/posts/:id` — удалить пост (админ)

## Технологии

**Бэкенд:**
- Node.js + Express
- SQLite (better-sqlite3)
- JWT для авторизации
- bcryptjs для хеширования паролей

**Фронтенд:**
- React 19
- React Router DOM v7
- Axios для API запросов
- CSS3 с градиентами и анимациями

- Клиент: http://localhost:3000
